Get up and running with the YouVersion Platform API in minutes.

## Prerequisites

Before you begin, you'll need:

- A YouVersion developer account
- Basic knowledge of HTTP requests
- A programming language of your choice (we'll show examples in multiple languages)

## Step 1: Create Your Developer Account

Sign in to YouVersion and create your developer account at [login.youversion.com](https://login.youversion.com)

## Step 2: Create an App ID

To use the YouVersion Platform API, you need to register your app and obtain an `X-App-Id` (app id).

Visit [developers.youversion.com/get-started](https://developers.youversion.com/get-started) to create an account and register your application.

## Step 3: Make Your First Request

Get up and running with the YouVersion Platform API in minutes.

### Using cURL

```bash
curl -H "X-App-Id: YOUR_APP_ID" https://api-dev.youversion.com/v1/bibles/206/books/jhn/chapters/3/verses/16
```

### Using JavaScript (Node.js)

```javascript
const fetch = require('node-fetch');

async function getVerse() {
  const response = await fetch('https://api-dev.youversion.com/v1/bibles/206/books/jhn/chapters/3/verses/16', {
    headers: {
      'X-App-Id': 'YOUR_APP_ID'
    }
  });
  
  const data = await response.json();
  console.log(data);
}

getVerse();
```

### Using Python

```python
import requests

def get_verse():
    headers = {
        'X-App-Id': 'YOUR_APP_ID'
    }
    
    response = requests.get('https://api-dev.youversion.com/v1/bibles/206/books/jhn/chapters/3/verses/16', headers=headers)
    data = response.json()
    print(data)

get_verse()
```

## Step 5: Understand Authorization

All API requests require your app key in the `X-App-Id` header.

```bash
X-App-Id: YOUR_APP_ID
```

## Step 6: Learn About Authentication

Requests for user's information require sign in with YouVersion.

### Authentication Flow

1. **User Authentication**: Users sign in with their YouVersion account
2. **Token Generation**: A Limited Access Token (LAT) is generated
3. **API Access**: Use the LAT for authenticated requests

### Example Authenticated Request

```bash
curl -H "X-App-Id: YOUR_APP_ID" \
     -H "Authorization: Bearer USER_LAT_TOKEN" \
     https://api-dev.youversion.com/v1/user/profile
```

## Common API Endpoints

### Get Bible Versions

```bash
curl -H "X-App-Id: YOUR_APP_ID" \
     https://api-dev.youversion.com/v1/bibles
```

### Get Books for a Bible

```bash
curl -H "X-App-Id: YOUR_APP_ID" \
     https://api-dev.youversion.com/v1/bibles/206/books
```

### Get Chapter Content

```bash
curl -H "X-App-Id: YOUR_APP_ID" \
     https://api-dev.youversion.com/v1/bibles/206/books/mat/chapters/1/verses
```

### Search Bible Content

```bash
curl -H "X-App-Id: YOUR_APP_ID" \
     "https://api-dev.youversion.com/v1/bibles/206/search?q=love"
```

## Error Handling

Always handle potential errors in your code:

```javascript
async function getBibles() {
  try {
    const response = await fetch('https://api-dev.youversion.com/v1/bibles', {
      headers: {
        'X-App-Id': 'YOUR_APP_ID'
      }
    });
    
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Error fetching bibles:', error);
  }
}
```

## Common Parameters

### Language Ranges

When requesting Bible versions, you can specify language preferences:

```bash
curl -H "X-App-Id: YOUR_APP_ID" \
     "https://api-dev.youversion.com/v1/bibles?language_ranges=en,es,fr"
```

### Pagination

Many endpoints support pagination using the `next_page_token` parameter:

```bash
curl -H "X-App-Id: YOUR_APP_ID" \
     "https://api-dev.youversion.com/v1/bibles?next_page_token=2"
```
